# Laboratorios de Ingeniería de Software II – 2026-1

¡Bienvenidos al ciclo 2026-1!

Este repositorio contiene los laboratorios que trabajaremos a lo largo del curso. Durante el ciclo, desarrollaremos un caso integrador llamado **MercaGo**, una empresa retail que se encuentra en proceso de transformación digital y necesita construir, mejorar y escalar su sistema de ventas.

La idea es que cada laboratorio represente una etapa distinta de la evolución del sistema. De esta manera, no trabajaremos ejercicios aislados, sino un caso progresivo y cercano a situaciones reales del entorno profesional.

## Caso integrador: MercaGo

A lo largo de estas sesiones, asumiremos el rol de desarrolladores de software dentro del equipo de MercaGo. Conforme avance el ciclo, iremos resolviendo nuevas necesidades del negocio: modelado de productos, carrito de compras, pagos, mejora de la calidad del código, incorporación de patrones de diseño e introducción a la planificación ágil.

## Objetivo del repositorio

Este repositorio tiene como propósito:

- compartir los laboratorios y materiales prácticos del curso;
- organizar el trabajo por sesiones o temas;
- reforzar los conceptos vistos en clase mediante ejercicios aplicados;
- mostrar cómo un sistema puede evolucionar progresivamente desde una versión básica hasta una solución mejor estructurada.

## Secuencia de laboratorios

### Lab. 01 – Ejercicios de POO, herencia, clases abstractas y polimorfismo (Java)
En esta sesión construiremos la primera versión del sistema de ventas de MercaGo. Modelaremos productos, carrito de compras y métodos de pago aplicando Programación Orientada a Objetos.

### Lab. 02 – Ejercicios de violación de principios SOLID (Java)
Una vez construido el sistema base, analizaremos partes del código que presentan problemas de diseño. El objetivo será refactorizar y mejorar la estructura del sistema aplicando los principios SOLID.

### Lab. 03 – Ejercicios de patrones creacionales (Java)
En esta etapa trabajaremos cómo crear objetos de forma más ordenada, flexible y mantenible, aplicando patrones creacionales dentro del sistema de MercaGo.

### Lab. 04 – Ejercicios de patrones estructurales (Java)
Aquí abordaremos problemas de integración con servicios externos y simplificación de módulos internos, aplicando patrones estructurales para mejorar la arquitectura del sistema.

### Lab. 05 – Ejercicios de patrones de comportamiento (Java)
En esta sesión organizaremos mejor las acciones y reglas del sistema, aplicando patrones de comportamiento para reducir duplicación y facilitar futuras extensiones.

### Lab. 06 – Diseño Ágil
Finalmente, trabajaremos la planificación de mejoras del sistema desde una perspectiva ágil, utilizando historias de usuario, backlog y priorización de funcionalidades.
